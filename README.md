## Formatting

Format your PR titles likewise:

> [Tag]: Describe changes
Tag can be:

- Enhance (new feature)
- Error (bug fix)
- Refactor (refactoring code)
- Style (formatting, no code change)
- Doc (changes to documentation)
- CMS (change to CMS page)
